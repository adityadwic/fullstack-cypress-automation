# Contributing to UI Automation Framework

We welcome contributions to improve this framework!

## Development Process
1.  **Fork** the repo on GitHub.
2.  **Clone** the project to your own machine.
3.  **Commit** changes to your own branch.
4.  **Push** your work back up to your fork.
5.  Submit a **Pull Request** so that we can review your changes.

## Code Standards
- **Page Object Model**: All selectors and methods interacting with the UI must reside in a Page Class.
- **Strict Linting**: Ensure `eslint` passes before pushing.
- **Type Safety**: Avoid `any` types; define interfaces for data structures.

## Commit Messages
Please follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:
- `feat: add user profile page object`
- `fix: correct login timeout`
- `chore: update dependencies`
